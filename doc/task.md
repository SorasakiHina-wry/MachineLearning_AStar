# 算法研讨任务流参考文档

## 1. 目标
1. 研讨核心课题为机器学习思想及算法在A\*搜索中的应用。
2. 具体解决问题为：在图的启发值提供不完备的情况下，建立一个模型可对启发值进行预测，使得A\*算法可以应用在这样的图搜索上。（不保证预测启发值可参考）
3. 最终交付物包括以下：
- [ ] 一个完整的C++项目，包含模型训练部分与模型应用部分，以及测试程序及数据
- [ ] 用于汇报的展示性幻灯片
- [ ] 关于项目与算法的研究报告

## 2. 任务拆解
### 1.程序项目
#### 理论部分
- [ ] 学习解析机器学习的基础知识
- [ ] 总体上排布好整个项目的结构与线路
#### 代码部分
*基于面向对象的思想，独立解耦地开发各个需要的类，最终进行并入耦合*
- [ ] 配置参数的读取与传入
- [ ] 图数据读取与矩阵格式化
- [ ] A*算法的接口
- [ ] ***核心*** 模型训练类
- [ ] ***核心*** 模型应用类
- [ ] 最终结果的序列化与输出

### 2. 研讨报告
*按照常规的报告撰写结构，重点在于机器学习部分的算法设计与分析*
- [ ] 整体结构与各部分模块的结构组织
- [ ] 学习算法的设计思路
- [ ] ***核心*** 学习算法的行进逻辑与功能

### 3. 课题展示
*以幻灯片作为展示平台*
- [ ] 课题的选择目的与思路
- [ ] 整体的行进路线
- [ ] 学习算法的具体细节
- [ ] 实现细节上的性能优化方案
- [ ] 算法复杂度分析

## 3. 路线规划
1. 研究理解简单机器学习的算法思路本质与实现思想
2. 产出一个简单的原型版本，作为基准与参考
3. 分析性能瓶颈
4. 理论与实践上同步优化算法细节
5. 对比各版本的性能表现

## 4. 工具选择
1. 编程语言: C++20
2. 代码风格：类名遵循驼峰命名法(CamelCase)，函数与变量遵循蛇形命名法(snake_case)，类数据成员名以下划线结尾(snake_case_)
3. 使用Git版本控制，在Github同步合并进度
4. 报告模板统一用基于markdown的LaTeX模板
